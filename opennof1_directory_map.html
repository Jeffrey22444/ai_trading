# opennof1 项目目录地图（给人看的版本，不含缓存/依赖噪音）
## 1. 总体定位
这是一个 AI 自动交易系统，但当前工程重点不是“支持很多交易所”，而是把一条可安全验证的单一路径跑通。
当前主路径：DeepSeek 负责决策，Hyperliquid Testnet 负责行情和下单，前端负责观察与控制。

├── README_zh.md ← 中文总入口

## 3. 你最应该理解的 3 条主链路
3.1 行情链路 HyperliquidMarketClient → data_cache / derivatives_cache → quant / API / Agent
3.2 量化链路 quant/scoring + stops + position_sizing → quant_guardrail → AI prompt / 持仓退出兜底 / execution
3.3 决策链路 analysis_node → trading_execution_node → save_analysis_node → frontend DecisionsList
3.4 prompt 链路 SQLite 运行时策略 → trading_strategy.md 初始化/Reset 模板
## 4. 当前阶段真正的重点
重点 1: 把 Hyperliquid testnet 路径跑稳，而不是继续维护 Binance 等旧路径。
重点 2: 确保自动交易不会误启动，所以 scheduler 需要显式 start，默认 live trading 关闭。
重点 3: prompt 可以在前端运行时修改并落 SQLite；Git 默认模板在 trading_strategy.md。
重点 4: 策略 v2 的评分、Kelly、止损、持仓退出不靠 AI 猜，必须由 quant 代码和配置支撑。
重点 5: 最终验收不是“页面能打开”就算完成，而是看 pytest、前端 build、p0_acceptance.py 和本地 runbook。
## 5. 新手阅读顺序建议
1. LOCAL_RUNBOOK.md 先知道怎么跑、怎么验收
2. STRATEGY_V2_CONFIG_GUIDE.md 理解评分权重、Kelly、止损参数怎么调
3. backend/agent/quant/guardrails.py 理解代码如何汇总量化护栏
4. backend/api/routes.py 再看系统暴露了哪些能力
5. backend/services/strategy_contract.py 确认策略文字能引用哪些字段
6. frontend/src/components/trading/DecisionsList.tsx 最后看前端怎么展示量化决策

关键入口

  • backend/api/main.py：后端 API 启动入口
  • frontend/src/app/page.tsx：仪表盘总装页面
  • backend/agent/quant/guardrails.py：策略 v2 量化护栏汇总入口
  • STRATEGY_V2_CONFIG_GUIDE.md：未来调参操作指南
  • backend/scripts/p0_acceptance.py：最终验收脚本

当前架构结论

  • 运行时已经收敛到 Hyperliquid perpetuals only
  • 行情来源是 CCXT REST polling，不是 WebSocket 实时推送
  • 策略 v2 的仓位、止损、止盈、杠杆由代码护栏约束
  • 策略支持前端运行时编辑，并落 SQLite；Reset 会恢复 trading_strategy.md 模板，字段引用受 strategy_contract.py 校验

最容易误解的点

  • 启动后端 != 已经开始自动交易
  • 改前端策略 != 改了 Git 里的默认模板；要更新默认模板需改 trading_strategy.md
  • 改 prompt != 改了 D1-D5 评分规则，规则改动要同步 quant 代码和测试
  • 目录里很多缓存和依赖，不等于你真正需要读的代码
